# 设备数据统计分析系统

## 项目概述

Aopeng 生产设备合格率统计与分析系统。本项目用于分析和统计设备生产数据中的合格/不合格情况，包括时长统计、审批状态跟踪和日期/设备维度的详细分析。

**主要指标：**
- 已审批-有效数
- 已审批-无效数  
- 待审批数
- 总时长（分钟/小时）
- 待审批时长
- 合格时长
- **合格率（%）**

## 审批状态说明

| 状态代码 | 含义 | 说明 |
|---------|------|------|
| 已审批 | 审批通过的有效数据 | 计入合格时长 |
| 已审批-无效片段过长 | 审批通过但质量不达标 | 计入不合格时长 |
| 待审批 | 待审批的数据 | 统计但不计入合格率 |

## 合格率计算逻辑

```
合格时长 = (原始上送时长 - 无效时长) for ALL 已审批记录
不合格时长 = 所有无效时长之和（包含"已审批-无效片段过长"）
合格率(%) = 合格时长 / (合格时长 + 不合格时长) × 100
```

**3月23日-4月10日统计结果：**
- 合格时长：298.85 小时
- 不合格时长：83.77 小时  
- **总体合格率：78.1%**

## 核心脚本说明

### 1. analyze_by_date_13.py
按日期统计，生成各日期的合格率、问题数等指标。

**输入：** QAQ.csv
**输出：** 13台设备统计_按日期_3月23日-4月10日_修正.csv

**主要字段：**
- 日期
- 已审批数
- 已审批无效数
- 待审批数
- 总计
- 合格时长
- 不合格时长
- 合格率(%)

### 2. analyze_by_device_13.py
按设备统计，生成各设备的合格率排名。

**输入：** QAQ.csv
**输出：** 13台设备统计_修正版_3月23日-4月10日.csv

**主要字段：**
- 设备ID
- 已审批数
- 已审批无效数
- 待审批数
- 合格率(%)

### 3. generate_excel_from_csv_corrected.py
将 CSV 文件转换为格式化的 Excel 文件。

**功能：**
- 条件格式化：≥85% 绿色、70-85% 黄色、50-70% 浅红、<50% 红色
- 自动调整列宽
- 添加边框和蓝色标题行

**用法：** 运行脚本自动查找并转换所有 CSV 文件

### 4. extract_problem_devices_0202_0331.py
提取指定设备在特定时间范围内视频有问题的数据。

**功能：**
- 从 QAQ.csv 筛选指定设备和时间范围
- 只提取"已审批-无效片段过长"和"待审批"状态的数据
- 生成详细数据表和每日统计表

**输入：** QAQ.csv  
**输出：** 
- `8台设备问题数据_02-02到03-31.csv` - 1,525 条详细记录
- `8台设备每日统计_02-02到03-31.csv` - 53 天每日统计

**运行示例：**
```bash
python scripts/extract_problem_devices_0202_0331.py
```

**配置参数：**
```python
DEVICES_TO_EXTRACT = ['2f59', 'ee38', '22ad', '5683', '4f53', '082c', '1af8', 'd8db']
START_DATE = '2026-02-02'
END_DATE = '2026-03-31'
PROBLEM_STATUSES = ['已审批-无效片段过长', '待审批']
```

### 5. convert_problem_data_to_excel.py
将问题数据 CSV 转换为格式化的 Excel 报告。

**功能：**
- 读取详细数据和每日统计 CSV
- 生成 Excel 文件（两个工作表）
- 应用统一的格式化和边框样式

**输入：** CSV 文件  
**输出：** `8台设备问题数据_02-02到03-31.xlsx`

**用法：**
```bash
python scripts/convert_problem_data_to_excel.py
```
