# 计算逻辑修正说明 - 2026年4月13日

## 📊 修正内容

### 原有问题
之前的统计只计算了"已审批"状态的有效时长，忽略了"已审批-无效片段过长"中包含的无效时长。

### 修正方案
将"已审批-无效片段过长"中的无效时长也统计为不合格时长，重新计算合格率。

**修正前的逻辑:**
- 有效时长 = 已审批的（原始上送时长 - 无效时长）
- 不合格时长 = 已审批-无效片段过长的总时长（错误）
- 合格率 = 有效时长 / (有效时长 + 不合格时长)

**修正后的逻辑:**
- 有效时长 = 已审批的（原始上送时长 - 无效时长）+ 已审批-无效的（原始上送时长 - 无效时长）
- 不合格时长 = 已审批的无效时长 + 已审批-无效的无效时长
- 合格率 = 有效时长 / (有效时长 + 不合格时长)

elif status == '已审批-无效片段过长':
    date_stats[date_key]['approved_invalid'] += 1

**合格率计算阶段:**
```python
# 修正前

```
